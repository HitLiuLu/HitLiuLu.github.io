开发中gulp、bower以及npm的一些便捷的使用

使用angularjs开发webapp时，很多工具可以提高我们的开发效率，降低我们的开发复杂度。其中自动化构建工具gulp可以增强工作流程，帮助我们快速构建项目并减少频繁的IO操作；包依赖管理工具bower可以帮助我们方便的安装开发中要是用的js类库，并且自动下载该类库所依赖的包；而npm作为node的包管理工具，同样可以帮助我们安装第三方包，除此之外，它还可以生成package.json文件，该文件配置了项目所需运行的运行和开发环境，定义了项目所需的各种模块。下面我们将记录它们的一些简单的web开发中的一些常用使用。

1. A package manager for the web - Bower

顾名思义，bower是一款web开发的包管理工具。很多说法也称它为js包依赖管理工具，因为它可以很容易地管理js库之间的依赖，并自动安装这些依赖。举个例子，之前前端开发要引入bootstrap，由于bootstrap是需要依赖jquery的，我们就要先去引入一个jquery的cdn，然后再引入一个bootstrap的cdn。前端框架何其多，如果要引入的框架所需的依赖我们并不清楚或疏忽了，就会直接搞砸我们的开发。而使用了bower引入bootstrap呢，bower会自动安装它所需要依赖的jquery依赖版本，真的是轻松方便又享受啊。

配置文件中的字段表明了这个web项目一些信息。

接下来，假设我们开发中要用到bootstrap，需要我们在控制台中敲击命令：bower install bootstrap --save，其中--save表示在bower.json中添加配置信息：

我们发现bootstrap被添加到了项目中的依赖列表里。

如果我们不需要Bootstrap了，就可以通过命令bower uninstall bootstrap --save卸载bootstrap依赖。同理，若需要保存依赖配置信息，去掉--save即可。

2.npm - npm is the package manager for JavaScript

NPM是随同NodeJS一起安装的包管理工具，它允许用户从NPM服务器下载别人编写的第三方包到本地使用，也允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

本文所介绍的bower和gulp都是通过它安装使用的：
npm install bower -g和npm install gulp -g

同样的，我们在使用npm管理整个项目，开始，要是用npm init命令初始化，并生成package.json配置文件：

{
  "name": "web_test",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
  }
}

这个文件定义了项目所需要的各种模块，以及项目的配置信息（比如名称、版本、许可证等元数据）。
然后这是我在项目完成后，package.json文件的信息：

{
  "name": "web_test",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "gulp": "^3.9.1",
    "gulp-clean": "^0.3.2",
    "gulp-concat": "^2.6.1",
    "gulp-connect": "^5.0.0",
    "gulp-cssmin": "^0.1.7",
    "gulp-imagemin": "^3.1.1",
    "gulp-less": "^3.3.0",
    "gulp-load-plugins": "~1.5.0",
    "gulp-plumber": "^1.1.0",
    "gulp-uglify": "^2.0.1",
    "open": "^0.0.5"
  }
}

我们发现较之前的文件，devDependencies字段的对象更加丰富了，这个字段指定了项目开发所需要的模块。她指向的对象的成员分别由模块名和版本要求组成，表示依赖的模块和版本范围。当我们拿到package.json文件时，直接使用npm install命令即可安装项目的这些依赖模块。利用npm install gulp --save-dev就可以写入devDependencies字段中了。

script字段指定了运行脚本命令的npm命令行缩写，更为具体的例子：

"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1",
  "webpack": "node_modules/.bin/webpack",
  "start": "webpack-dev-server --hot --inline",
  "build": "node_modules/.bin/webpack --progress --profile --colors --config webpack.production.config.js"
}

例如start指定了运行npm run start时所要执行的命令。

“gulp”: “^3.9.1” 我们看到属性值为gulp的值为^3.9.1，这里的^表示安装3.x.x的最新版本，但是不安装2.x.x也就是安装时不改变大版本号，
“gulp-load-plugins”: “~1.5.0”,波浪号表示1.5.x的最新版本，但是不安装1.6.x，也就是说安装时不改变大版本号和次要版本号。

暂时用到的字段比较少，剩下的留有日后补充。

值得注意的是，由于在速度和被墙等方面的原因，推荐使用淘宝在国内的npm镜像，即cnpm，使用起来也很简单，通过npm install -g cnpm安装后，以后在使用到npm的地方统统换成cnpm就行了。例如：

npm install gulp 
cnpm install gulp

他俩是等价的。

3.gulp - Automate and enhance your workflow

顾名思义，gulp作为一款自动化构建工具，可以增强工作流程，提升工作效率。实际上，gulp是基于Nodejs的自动任务运行器， 她能自动化地完成 javascript/coffee/sass/less/html/image/css 等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。应用管道（pipe）的思想，前一级的输出，直接变成后一级的输入，使得在操作上非常简单。下面简单记录项目中gulp的一些简单使用。
首先是安装，上文介绍npm时已经有所提及，首先是npm install gulp -g，然后在开发目录下使用命令行npm install --save-dev gulp将gulp模块添加到配置文件package.json中。

安装完毕，新建gulpfile.js文件,它是gulp项目的配置文件，下面是已经写好的一个gulpfile.js文件：

var gulp = require('gulp'); //导入gulp工具包

var app = {
    srcPath: 'src/',
    devPath: 'build/',
    prdPath: 'dist/'
};

gulp.task('lib', function(){  
    gulp.src('bower_components/**/*')
    .pipe(gulp.dest(app.devPath + 'vendor'))
    .pipe(gulp.dest(app.prdPath + 'vendor'))
});


gulp.task('default', ['lib']);

gulp.src(path)表示输出（Emits）path中地址的文件。 它将返回一个 Vinyl files 的 stream 它可以被 piped 到别的插件中。
gulp.dest(path)能被 pipe 进来，并且将会写文件。并且重新输出（emits）所有数据，因此你可以将它pipe到多个文件夹。如果某文件夹不存在，将会自动创建它。

下面我们看一则在gulp中使用插件的例子：

var gulp = require('gulp'), 
    less = require('gulp-less');

gulp.task('testLess', function () {
    gulp.src('src/less/index.less') 
    .pipe(less()) 
    .pipe(gulp.dest('src/css')); 
});

gulp.task('default',['testLess', 'elseTask']);

本段代码表示在gulp中引入了gulp-less插件，新建任务testLess，这个任务首先调用gulp的api - src()方法定位文件，将其转化成流后pipe到插件的gulp-less，然后用它的.less()方法处理，最终将其pipe到dest文件夹中，生成最终文件。

gulp.task(name[, deps], fn)定义一个使用 Orchestrator 实现的任务（task）。其中，name指任务的名字，deps是一个包含任务列表的数组，这些任务会在你当前任务运行之前完成，可以理解为该任务依赖deps中的任务，执行该任务时需要提前实现依赖。任务运行时，即执行gulp命令时，首先执行default任务。
