less的基本使用

Less 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展，帮助我们更便捷的进行web开发。Less is more！
下面我将通过我在实际开发中less中的一些应用，记录并分享它的一些基本用法：

1.Less的运行环境

Less可以在多种环境中使用，包括浏览器端、桌面客户端、服务端等。这里我主要用到了两种方式开发Less，它们分别是使用桌面客户端koala和用gulp的插件gulp-less编译。

Koala

第一步，在koala官网中下载koala客户端并安装；

第二步，编辑less文件，在koala中打开此less文件；

第三步，点击要编译的less文件，选择执行编译即可编译成css文件；

gulp插件

当我们模块化开发一个较复杂的Web应用时，若还使用koala编译less文件就会变得特别麻烦

下图是在做的一个模块化开发的webapp时要用到的less文件：

这还只是一个特别简单的webapp的一部分而已，这么多less文件用koal手动编译在手动引入的话操实在过于繁琐。在使用自动化构建工具gulp时，发现一款插件gulp-less，它可以简化繁杂单调的编译工作，只需要在gulpfile.js里简单的配置几行代码而已。

var gulp = require('gulp');
var less = require('gulp-less')();

var app = {
    srcPath: 'src/',
    devPath: 'build/',
    prdPath: 'dist/'
};

gulp.task('less',function(){
    gulp.src(app.srcPath + 'style/index.less')
    .pipe(less.less()) //在任务中接收src pipe过来的流，编译less文件，将结果pipe到目标文件夹中
    .pipe(gulp.dest(app.devPath + 'css'))
    .pipe(gulp.dest(app.prdPath + 'css'));
});

有关gulp的详细用法可以参照我的上篇日记开发中gulp、bower以及npm的一些便捷的使用 或者转到更加权威的gulp官网。

2.less的一些常用语法

定义变量
比如颜色这种属性，我们通过定义变量再调用的方式设置颜色，变量名的定义方式是用@变量名：值;的方式，如下图：


上图这样的方式，当我们设置或改变皮肤时只需要改变颜色变量就可以了，非常的方便。看一个更加清晰的例子

@width:400px;
@height:300px;
@font_size:12px;

textarea {
    width:@width;
    height:@height;
    font-size:@font_size;
}

这段less代码编译后的css代码就是：

textarea {
  width: 400px;
  height: 300px;
  font-size: 12px;
}

后代选择器
后代选择器我的理解就是css类可以嵌套，看一个例子：

@width:400px;
@height:300px;
@font_size:12px;

textarea {
    width:@width;
    height:@height;
    font-size:@font_size;
    .abc {
        font-size:@font_size;
    }  
}

这段less代码编译后的css代码就是：

textarea {
  width: 400px;
  height: 300px;
  font-size: 12px;
}
textarea .abc {
  font-size: 12px;
}

也就是说在less中类嵌套类，编译出来就是一个类的后代类的样式。

值得注意的是,在后代选择器中使用&符号，不仅可以实现后代选择，还可以实现诸如:hover 和 :focus这样的伪类：

@width:400px;
@height:300px;
@font_size:12px;

a {
    width:@width;
    height:@height;
    font-size:@font_size;
    &:hover {
        font-size:@font_size;
    }  
}

编译后的css为：

a {
  width: 400px;
  height: 300px;
  font-size: 12px;
}
a:hover {
  font-size: 12px;
}

文件引用

文件引用的方式是在less文件中写入@import PATH;,PATH可以是相对路径。当我们在一个less文件中将样式需要的所有变量列出来时，我们在其他less文件中需要引入这些变量，这样我们就需要文件引用功能。文件引用更重要的一个应用是在模块化开发的时候，将一个web项目切分成若干个小模块，为了便于管理，我们将每个模块对应编写对应的样式文件，但当样式文件很多时，less文件的编译和引入就会变得异常繁琐，这时候我们可以利用文件引用功能将这些less文件全部导入到一个初试less文件中，这样就可以只编译、引入一次就可以完成以前相当繁琐的任务了。例如，我将编写的所有的模块less文件引入到index.less的总样式文件中去：

函数

函数特性是less中一个重要特性，它使得我们在编译样式的时候可以像编辑js一样动态地创建、调用样式。举个例子：

@width:400px;
@height:300px;

textarea {
    width:@width;
    height:@height;
    .fs(13px);
}

.fs(@px) {
  font-size: @px;
}

它编译后的css代码为

textarea {
  width: 400px;
  height: 300px;
  font-size: 13px;
}

不要忘记在less中变量前一定要加@符号。
