正则表达式--属性与方法

昨天学习了正则表达式的类那一部分，今天学习了剩下的部分。

量词

假如我们想匹配一个连续出现20次数字的字符串，根据上次所学，我们可以写一个正则表达式来匹配，即\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d\d，我们发现这样写非常麻烦，我们可以使用\d{20}来匹配。

  • ?：出现零次或一次（最多出现一次）
  • +：出现一次或多次（至少出现一次）
  • *：出现零次或多次（任意次）
  • {n}：出现n次
  • {n,m}：出现n到m次
  • {n,}：至少出现n次
  • {0,n}：至多出现n次

贪婪模式与非贪婪模式

贪婪模式：

'12345678'.replace(/\d{3,6}/g, 'X')
"X78"

JS默认进行贪婪匹配，所以会匹配6个数字，然后进行替换。

非贪婪模式：让正则表达式尽可能少的匹配，也就是说一旦成功匹配不再继续尝试就是非贪婪模式。非贪婪模式的做法是在量词后加上?即可。

'12345678'.match(/\d{3,5}?/g)
["123","456","789"]

分组

匹配字符串连续出现3次的场景，我们使用Byron{3}，我们发现匹配了3个字母n，这样我们就不能匹配连续出现的词组。使用()可以达到分组的功能，使量词作用于分组，即(Byron){3}

或

正则中使用|可以达到或的效果，Byron|Casper可以匹配Byron或Casper，Byr(on|Ca)sper可以匹配on或Ca。

反向引用

我们想将2015-12-25变换成12/25/2015，需要使用反向引用。

'2015-12-25'.replace(/(\d{4})-(\d{2})-(\d{2})/g, '$2/$3/$1')

这样就可以变换格式成功。

原理：将字符串分成3组，分别是group1,group2和group3，$1反向引用第一组，同理$2和$3也是如此。

忽略分组

不希望捕获分组时，只需要在分组内加上?:就可以。

(?:Byron).(ok)
//匹配到一个字符串加一个任意字符再加上一个分组(ok)

JS对象属性

  • lastIndex：是当前表达式匹配内容的最后一个字符的下一个位置
  • source：正则表达式的文本字符串
    再加上之前说过的三个修饰符，分别对应的三个属性：global、ignore和multiline

test和exec方法

  • test用于测试字符串参数中是否存在匹配正则表达式模式的字符串，如果存在返回true，不存在返回false。

    var reg1 = /\w/;
reg1.test('a');
//true

  • exec使用正则表达式模式对字符串执行搜索，并将更新全局RegExpduixiang对象的属性以反映匹配结果，如果没有匹配的文本则返回null,否则返回一个结果数组：

    • index声明匹配文本的第一个字符的位置
    • input存放被检索的字符串string

字符串对象方法

  • search()方法用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串。
