对nth选择器的误解分析

因为要做一个以页面样式为主要工作的官网，昨天开始系统地学习CSS3的一些知识。首先接触的就是css选择器，发现之前对css选择器的理解一直不够准确，存在误解，现纪录下来。

nth选择器

我们都知道nth选择器可以用于定位元素位置，从而给其设置相应的样式。通过对它的学习总结，我认为nth选择器可以大概分为两个阵营。其一我把它称为child阵营，其二我把它叫做of-type阵营。下面分别说明我为什么这么分。

child阵营

这个阵营哪些成员，又有何共同、特有特征呢。下面先总结：

  • :first-child: 匹配其父元素的第一个元素
  • :last-child: 匹配其父元素的最后一个元素
  • :nth-child(N): 匹配其父元素的第N个元素
    其中N可以是具体数字也可以是关于n的表达式，例如:nth-child(1)就和:first-child实现的功能相同，都是选择其父元素的第一个元素。
    :nth-child(2n+1)表示取奇数位元素，:nth-child(2n)表示取偶数位元素，分别相当于:nth-child(odd)和:nth-child(even)。
  • :nth-last-child(N): 表示从最后一个元素开始计数，值得注意的是，css选择器中的数都是从1开始计的，这与js中的以0开始的计数规则不同。

在介绍特征之前先说一下我以前对它的误解吧，看一个例子：

<html>
<head>
    <title>child</title>
    <style>
        section:first-child {
            color: #f00;
        }
    </style>
</head>
<body>
    <section>
        <div>1-1</div>
        <div>1-2</div>
        <div>1-3</div>
    </section>
    <section>
        <div>2-1</div>
        <div>2-2</div>
        <div>2-3</div>
    </section>
</body>
</html>

按照我之前的理解，section:first-child选择的是section下面的第一个子元素，所以就是1-1和2-1变红，我们看一下实际结果：

很明显，结果与我们想象的不同。因为以前的这种想法其实是误解了nth选择器。

接下来，我们归纳一下它们的共同特征。对于child阵营里的所有选择器来说，它们有两个特点值得我们注意。

    1. 其父元素：顾名思义，是:之前的元素的父元素下的子元素。
    1. 第N个: 以first-child为例，就是第一个子元素，当且仅当它是第一个时才会被选择。
      了解了这两个特点以后，在看上面的例子，首先第一点，是section的其父元素，及body元素，第二点就是body下面的第一个子元素若是section则选中，故变红的是1-1，1-2和1-3。

我们对上面的例子稍作改动，在section元素之前添加一个div：

<html>
<head>
    <title>child</title>
    <style>
        section:first-child {
            color: #f00;
        }
    </style>
</head>
<body>
    <div></div>
    <section>
        <div>1-1</div>
        <div>1-2</div>
        <div>1-3</div>
    </section>
    <section>
        <div>2-1</div>
        <div>2-2</div>
        <div>2-3</div>
    </section>
</body>
</html>

结果如下：

我们看到没有元素变红，这是因为不满足第二点的当且仅当条件，此时body子元素的第一个子元素是div，不是section，故不能选中。

我们拿:first-child作为实例，其它child阵营选择器与此同理。

of-type阵营

这个阵营哪些成员，又有何共同、特有特征呢。下面先总结：

  • :first-of-type: 匹配其父元素的特定类型的第一个元素
  • :last-of-type: 匹配其父元素的特定类型的最后一个元素
  • :nth-of-type(N): 匹配其父元素的特定类型的第N个元素
    其中N可以是具体数字也可以是关于n的表达式，例如:nth-of-type(1)就和:first-of-type实现的功能相同，都是选择其父元素的第一个元素。
    :nth-of-type(2n+1)表示取奇数位元素，:nth-of-type(2n)表示取偶数位元素，分别相当于:nth-of-type(odd)和:nth-of-type(even)。
  • :nth-last-of-type(N): 表示从最后一个元素开始计数，值得注意的是，css选择器中的数都是从1开始计的，这与js中的以0开始的计数规则不同。

下面主要看一下of-type与child的差别，其它不赘述。

<html>
<head>
    <title>child</title>
    <style>
        section:first-of-type {
            color: #f00;
        }
    </style>
</head>
<body>
    <div></div>
    <section>
        <div>1-1</div>
        <div>1-2</div>
        <div>1-3</div>
    </section>
    <section>
        <div>2-1</div>
        <div>2-2</div>
        <div>2-3</div>
    </section>
</body>
</html>

结果是：

还是和上面一样的结构，不同的是选择器将child换成of-type，我们发现section下的文字变红，因为of-type选择器选择的是特定类型，所以选择的是body下面section元素的第一个，忽略了其他元素。
